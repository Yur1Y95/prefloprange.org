# Болванка: парс нового лимита из GTO Wizard

Готовый промпт для нового чата, чтобы залить префлоп-диапазоны нового лимита
(NL200, NL400, …) в свой пак через браузер. Метод отработан на NL10/NL50/NL100
(2026-06-17); детали — `docs/problems.md` P-034 + `CLAUDE.md` решение #13.
Меняются только лимит и числа — логика 1:1.

---

## Вставь это в начало нового чата (пример — NL200)

> **Track — GTOWNL200: парс NL200 6-max 100bb из GTO Wizard в новый пак.**
> Метод полностью отработан на NL100 — повторяем 1:1.
>
> Сначала прочитай: `docs/roadmap.md` (абзацы «GTOWNL100» и «GTOWNL50» +
> «Импорт-инструментарий»), `docs/problems.md` P-034 — ОБЯЗАТЕЛЬНО блок
> «Уточнение по NL100-сессии (2026-06-17)»: там рабочий рецепт включения
> EV-вкладки и ловушки 5/6/7 (дефолтная глубина 300bb, Chrome «(1)», поллинг
> по барам AA), не переоткрывай их заново. `CLAUDE.md` решение #13 (грузится сам).
>
> **Предусловия** (я уже подготовил): солв NL200 6-max 100bb, дерево «With cold
> calls 2.5x», вкладка «стратегия + EV». Залогинен с доступом к NL200. Браузер
> (Chrome MCP) подключён. Запроси песочнице доступ: `request_cowork_directory ~/Downloads`.
> Блок множественных загрузок Chrome на NL100/NL50 не срабатывал; если попросит
> «разрешить скачивать несколько файлов» для `app.gtowizard.com` — скажи, разрешу одним кликом.
>
> **Setup пака:**
> - Считай с ЖИВОГО NL200-URL: `gametype`, `gmff_rake` и ВСЕ размеры дерева
>   (опен / 3-бет / 4-бет). **НЕ бери числа от NL100/NL50/NL10** — рейк другой →
>   все сайзы другие, перечитывай с нуля. `gametype` у каждого стейка свой
>   (у NL50 он вообще не похож на NL100) — НЕ угадывай, читай из URL.
> - **Дефолтная вкладка GTOW может открыться на 300bb — форсить `depth=100` в URL**
>   и сверять header («Cash 100бб») перед съёмом (ловушка-5 P-034).
> - Создай `data/GTOWNL200.json` (meta: Cash / 6max / 100bb, label
>   «GTO Wizard NL200 6-max 100bb»; config.positions = UTG/MP/CO/BTN/SB/BB,
>   rfi_positions = первые 5; spots/ev пустые `{}`), папку `tools/gtow_dumps/nl200/`,
>   и строку в `tools/gtow_dumps/limits.json`:
>   `"nl200": { "folder": "tools/gtow_dumps/nl200", "pack": "data/GTOWNL200.json" }`.
>
> **Scope:** всё за раз — RFI 5 + vs_RFI 15 + vs_3bet 15 + vs_4bet 15 = 50 узлов (метод надёжен).
>
> **Метод** (детали — P-034):
> - URL: `…&gametype=<с живого URL>&depth=100&gmff_rake=<с живого URL>&gmff_opening_size=25x&preflop_actions=<LINE>&history_spot=<N>`.
>   `<LINE>` — действия по местам через «-» (`F` / `R<size>` / `C`).
>   `history_spot`: RFI = seat(hero), vs_RFI = seat(hero), vs_3bet = 6, vs_4bet = 7.
>   Активная карта (`[data-tst$="_active"]`) = hero — сверяй перед сохранением.
> - **EV-вкладка:** каждая полная навигация сбрасывает режим в plain (URL-параметры
>   SPA игнорит). Включай EV по узлу JS-диспатчем полных mouse-событий
>   (pointerdown→mousedown→pointerup→mouseup→click) сперва по каретке дропдауна
>   «стратегия» (`.gw_pop_menu_opener` у вкладки, rect ≈ [85,179,14,18]), затем по
>   пункту `.gwbtns_btn` «стратегия + EV». НЕ координатный клик и НЕ `element.click()`.
>   Перед сохранением проверяй `document.querySelectorAll('.rtc_strategy_ev_range_normalized').length === 169`.
> - **Поллинг сохранения (ловушка-7):** жди ОДНОВРЕМЕННО ≥169 клеток + активную
>   карту + НЕпустые бары в клетке AA (`#0_AA` имеет не-синий `rgb()` в
>   `background-image`), иначе при быстрых навигациях сохранишь полупустой DOM
>   (0 рук / спот не детектится). Только потом включать EV и сохранять.
> - Сайзы читай «бесплатно» по ходу: порядок RFI → vs_RFI → vs_3bet → vs_4bet;
>   на узле vs_RFI кнопка рейза hero (`sab_btn_name_bet_big`) = 3-бет-сайз для
>   vs_3bet-линии, на vs_3bet — 4-бет-сайз для vs_4bet (грепом из дампов:
>   `grep -oE 'sab_btn_name_bet_big">Raise [0-9.]+' dump.html`).
> - Сохраняй страницу (`outerHTML`→Blob, `a.download`) в `~/Downloads`; **давай
>   каждому узлу уникальное имя** (Chrome дописывает «(1)» к повтору — ловушка-6).
>   Парс-верифицируй каждый дамп (`python3 tools/gtow_parse.py <file>`), потом
>   `cp` в `tools/gtow_dumps/nl200/` (только свежие).
> - Импорт: `python3 tools/gtow_import_limits.py --limit nl200 --apply`, следи за ⚠ SHRINK.
>
> **Готово =**
> - `python3 tools/gtow_coverage.py data/GTOWNL200.json` → 0 осталось (RFI/vs_RFI/vs_3bet)
>   + `spots.vs_4bet` = 15 пар; `ev` зеркалит `spots` 1:1;
> - тесты зелёные: `tools/test_gtow_tools.py`, `tools/test_gtow_parse.py`,
>   `test_drill_ev.py`, `test_srs.py`, `test_ranges_api.py`;
> - обнови `roadmap.md` (абзац GTOWNL200), при необходимости P-034/CLAUDE.md;
> - просканируй дампы на секреты и напомни проверить `git diff` перед push;
> - подчисти временные дампы из `~/Downloads` после импорта.

## Почему болванка так устроена
- **Трек + лимит одной строкой** — модель сразу понимает задачу (один чат — одна задача).
- **Что прочитать** — метод и 7 ловушек уже записаны (P-034), не переоткрываем с нуля.
- **«Не бери числа от NL100»** — главная содержательная ошибка: другой рейк → другие
  размеры опена/3-бета/4-бета, и сам `gametype` у каждого стейка свой. Всё перечитывать с живого URL.
- **Дефолт 300bb** — для NL100+ вкладка открывается на 300bb; форсить `depth=100`.
- **Scope явно** — полный префлоп-трип 50 узлов; на NL100 уверенно прошёл за один чат.
