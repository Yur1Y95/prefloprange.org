# Концепт: Главная страница (Home / Dashboard) — PreflopRange

> Статус: **черновик-концепт** (для обсуждения), 2026-06-19.
> Связано: `docs/design.md` §1–2 (направление, токены), `docs/roadmap.md` Track B.6 (богатство через Higgsfield), `CLAUDE.md` #14/#17 (Higgsfield-пайплайн, chrome-редизайн).
> Формат: концепт без кода — структура, логика, тексты, спека. Реализацию заводим отдельной задачей в roadmap.

---

## 1. Проблема и цель

**Боль.** При любом заходе и при каждом F5 пользователь утыкается в модуль **Drill** — сразу стол с раздачей. Это утомляет и не даёт «откуда оттолкнуться». Причина — техническая и точечная (см. §7.1): в `static/index.html` модуль Drill единственный отрисован видимым, а роутера/стартового состояния нет вообще, поэтому перезагрузка всегда возвращает в один и тот же зашитый экран.

**Цель.** Ввести **Главную (Home)** как точку входа по умолчанию — оболочку-дашборд, с которой пользователь стартует взаимодействие с тренажёром. Совместить два подхода:

- **GTO Wizard — практика.** Дашборд как «центральный хаб»: статы, последние сессии, быстрый вход в нужный дрилл, сводка прогресса. (В GTOW dashboard переехал на кастомизируемый layout с виджетами и Session Summary — мы берём идею хаба, не копию.)
- **GipsyTeam — интертеймент/комьюнити.** Лента, которая обновляется и которую хочется полистать: «Урок дня», динамика, движуха. Чтобы можно было «замереть на секунду, пролистнуть, почитать».

**Принцип.** Home — это не ещё один режим в одном ряду с Drill/Editor, а **лендинг-дашборд-оболочка**, в которую попадаешь первой и из которой расходишься по модулям.

## 3. Hero-секция (бриф п.1)

**Роль.** Полноэкранный (≈`100vw × 100svh`) кинематографичный ролик в духе Higgsfield: casino-noir + энергия. Кликабельный.

**Тематика и раскадровка** (луп ~5–6 c, без жёстких склеек — атмосферный «дышащий» бэкграунд):

| Бит | t | Что в кадре | Смысл |
|-----|---|-------------|-------|
| 1 | 0–1.5 c | Золотой туз/карта-рубашка проявляется из черноты (эхо твоего эмбосс-спейда) | вход, бренд |
| 2 | 1.5–3 c | Над фелтом загорается ячейка-за-ячейкой голограмма матрицы **13×13** (зелёный+золото, как «нейроны») | **тренировка → запоминание** |
| 3 | 3–4.5 c | Камера ведёт по премиум-столу; золотые фишки растут стопками-лесенкой; восходящая золотая линия винрейта | **crushing limits** |
| 4 | петля | Оседаем на тёмном вью с местом под лого PreflopRange, плавно уходим в начало | бренд + CTA |

Ключевой ход: **матрица 13×13 — это реальный объект продукта**, не абстрактное казино. Ролик визуализирует именно нашу петлю «заучил диапазон → закрутил лимиты».

**Поведение.**

- Видео: `autoplay muted loop playsinline`, постер-кадр (первый фрейм) показывается до загрузки и как fallback.
- Клик по hero: если есть прогресс → «Продолжить тренировку» (ведёт в §3 Continue-таргет); если новый юзер → плавный скролл к модулям. Поверх — явные CTA-кнопки (см. §6).
- Звук: по умолчанию **off** (политика автоплея); опц. кнопка unmute — на твоё решение (поинт §9).
- `prefers-reduced-motion` → статичный постер вместо видео (доступность + перф).
- Перф: `preload` только постера, ленивая загрузка видео; целимся в лёгкий файл (см. §10).

**Ассеты — генерим через Higgsfield** (как стол/фишки, `CLAUDE.md` #14). Keyframes — Nano Banana Pro (2K, 16:9), затем image-to-video луп. Конкретные джобы/файлы — §10. Песочница не тянет CDN Higgsfield → готовое ты скачиваешь и кладёшь в `static/home_art/` (тот же поток, что с `hf_*.png`).

## 4. Дашборд (бриф п.2)

Совмещаем **практику** (GTOW) и **интертеймент** (GipsyTeam). Хорошая новость: под каждый блок **уже есть данные в проекте** — почти ничего не надо выдумывать.

### 4.1. Continue / «Продолжить» (B) — практика первой строкой

Одна крупная плашка-герой дашборда: что делать прямо сейчас.

- **Learn (SRS):** «Учить N карт» — N due-карт на сегодня. Источник: `srs_api` (due/new/learned).
- **Drill:** «Продолжить: Cash · UTG · RFI» — последний пак/спот. Источник: `history.json` (последняя запись: `spot`, `hero_position`, пак) + `data/`.
- Это GTOW-логика «центрального хаба»: убрать трение, сразу в дело.

### 4.2. Модули (C) — карточки режимов

Сетка карточек по числу режимов из топбара. Каждая: арт/иконка, название, одна строка пользы, мини-статус.

| Модуль | Одна строка | Мини-статус (источник) |
|--------|-------------|------------------------|
| **Drill** | Реакция на спот под таймер | точность сессии (`stats.json`) |
| **Learn** | Интервальное заучивание (SRS/FSRS) | «N due сегодня» (`srs_api`) |
| **Editor** | Свои диапазоны (главный платный крючок, `icp_and_pricing.md`) | число паков (`/api/ranges/list`) |
| **Visualizer** | Смотреть диапазоны матрицей | — |
| **Postflop** | Решения по типам оппонентов | — |
| **Analyzer** | Разбор истории рук | бейдж «Coming soon» |

Иерархия: **Drill + Learn** — крупные (основная петля), остальные мельче. Карточка: `--surface-2`, `--radius-lg`, `--sp-5` паддинг, hover → `--shadow-md` (паттерн «UI-карточки» из `design.md` §3).

### 4.3. Прогресс и динамика (D) — бриф «прогресс/динамика»

- **Шкала прогресса (mastery):** % выученных карт (`learned/total` из SRS), общая точность, **streak** (дни подряд). Крупное число + ярлык — паттерн `.lc-num/.lc-label` из `design.md` §3.
- **График динамики:** точность/винрейт по дням. Источник — `history.json` (есть `ts` + `correct` на каждую руку → агрегируем по дате). Спарклайн «accuracy за 14 дней», «сыграно сегодня / всего», «сильный спот / слабый спот» (из `stats.json` по спотам).
- **Лестница лимитов (энергия / crushing limits):** показать покрытые GTOW-паки (в `data/` уже есть NL10/NL50/NL100/NL200/NL500) как «покорённые лимиты» — лёгкая геймификация и прямой отыгрыш hero-нарратива.

Это и есть «интертеймент»: своя динамика, в которой приятно залипнуть.

### 4.4. Лента «Урок дня» / новости (E) — бриф «новости», GipsyTeam-вайб

- **Готовый контент уже есть:** `daily_lessons/*.md` — датированные «Уроки дня» (реально интересные, со ссылками-источниками). Это идеальная лента «почитать на секунду».
- Раскладка: верхний **«Урок дня»** крупно (заголовок + превью + дата), ниже — лента прошлых уроков и недельных заметок (`weekly-notes.md`, `status-update-*.md`).
- Позже — внешние покерные новости/борды (по аналогии с GipsyTeam-лентой) отдельным источником; **MVP — свой контент, его уже достаточно.**

## 5. Визуальный стиль

Берём существующую дизайн-систему `design.md` §2 — ничего нового изобретать не нужно:

- **Палитра:** графит (`--bg #0a0b0e`, `--surface-1/2/3`), золото-акцент **скупо** (`--accent #c9a84c`), фелт/золото — только в hero как «герой».
- **Типографика:** Hanken Grotesk; hero-дисплей `--fs-2xl 36px` + `--tracking-widest 4px`; заголовки секций `--fs-xl`; ярлыки `--fs-xs --tracking-wide`.
- **Сетка/отступы/радиусы/тени/motion:** шкалы `--sp-*`, `--radius-lg`, `--shadow-md/lg`, `--dur-* / --ease-out`.

Соответствие `design.md` §1 буквальное: **hero = «богатство GG»** (фелт, карты, золото, тактильность), **дашборд = «строгость GTOW»** (сетка, `--surface-2` карточки, золото точечно). Тема одна — тёмная, как весь апп.

Новые паттерны (мелочь): hero overlay-градиент (читаемость текста на видео), hover-elevation карточек, цвет спарклайна (`--accent` / `--correct`), бейджи «лестницы лимитов».

## 7. Встраивание в стек (как делать, без кода)

Стек: FastAPI + ванильный JS (без фреймворка), `/` отдаёт `static/index.html`, режимы — show/hide `<main>` по клику таба (`drill.js:269`).

### 7.1. Корень боли (точно)

- `index.html`: `#drillMode` — единственный `<main>` **без** `style="display:none"`, а таб Drill — `class="mode-tab active"`. Плюс `drill.js` на загрузке инициализирует дрилл (раздаёт руку).
- Роутера/персистентности нет → F5 всегда возвращает в этот зашитый разметкой экран.

### 7.2. Что меняем (спека)

- **`index.html`:** добавить `<main id="homeMode">` (видим по умолчанию); всем остальным `<main>`, **включая `#drillMode`**, проставить стартовый `display:none`; `.active` перенести с Drill на новый вход «Home» (отдельный таб или клик по лого — поинт §9).
- **`drill.js` (обработчик `.mode-tab`, ~стр. 269–283):** добавить ветку `homeMode` в show/hide и кейс `mode === 'home'`; дефолтный режим = `home`. Гейт авто-раздачи: не дилить руку, пока Drill реально не открыт (сейчас инициализируется на загрузке).
- **Роутинг (лёгкий, ванильный):** hash-роутинг (`#/`, `#/drill`, …) + правило **«F5 → Home»** (это и есть фикс). Опц. тумблер «открывать сразу последний модуль» — для хардкор-юзеров. (Старый `storage.js` — deprecated, не использовать; персистентность через текущий серверный поток / новый лёгкий localStorage-ключ только для UI-настройки.)
- **Новые фронт-файлы:** `static/home.js` + блок стилей в `style.css` (или `home.css`), по образцу существующих модулей.

### 7.3. Бэкенд (минимум нового)

- Дашборд почти весь собирается из **существующих** эндпоинтов: `/api/stats`, `/api/history`, `/api/srs/*`, `/api/ranges/list`.
- **Единственный реально новый кусок** — `/api/lessons`: маленький роутер (`lessons_api.py` в духе `srs_api.py`), который читает `daily_lessons/*.md` → список `{date, title, excerpt, html}`. (Опц. агрегатор `/api/home/summary`, чтобы не дёргать 4–5 ручек — по желанию.)
- Тесты как у вас: pytest на новый api, `node --check` на новый js.
