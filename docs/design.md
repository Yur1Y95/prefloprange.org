# Poker Trainer — Design System

Создан: 2026-05-28. Источник истины по визуалу. Этот документ описывает **целевую** дизайн-систему; код (`static/style.css`) подтягивается к ней постепенно в рамках Track B (см. `roadmap.md`). Пока документ опережает код — это нормально, он задаёт направление.

---

## 1. Направление

Сочетание двух референсов:

- **GGPoker — богатство «героя».** Стол и карты выглядят премиально и тактильно: насыщенный фелт с глубиной, выраженный деревянный rim, карты с чистой типографикой, мягкими тенями и аккуратной рубашкой, фишки с denominations. Это эмоциональный центр экрана.
- **GTO Wizard — строгость всего остального.** Интерфейс вокруг стола — дисциплинированный, сетка-выверенный, информационно-чистый. Приглушённая профессиональная палитра, один уверенный акцент (золото) используется скупо, резкая типографика, плотная читаемая матрица 13×13. Энергетика «серьёзный инструмент для изучения», не казино.

Правило компоновки: **роскошь концентрируется в столе и картах, всё остальное — сдержанно и строго.** Если сомневаешься, делать элемент богаче или строже — делай строже, кроме случаев когда это стол/карты/фишки.

**Фелт и стол (богатство GG — яркий казино-зелёный + чёрный рейл, по референсу CoinPoker, B.1-polish 2026-05-29).**

--felt-hi:     #379a59   /* светлый центр фелта */

Дерево/бордо/тил/графит рассматривались, выбран яркий зелёный + чёрный рейл.

## 3. Инвентарь компонентов

Каждый компонент — его назначение, анатомия, какие токены берёт. Цель: чтобы при редизайне (и при работе будущего дизайнера) был единый список «что вообще есть в системе».

**Кнопки.**
- *Primary* — золотая обводка + `--accent-bg` фон + `--accent` текст. Один primary на экран (Next, Учить, Save). Сейчас: `.learn-start-btn`, `.learn-next-btn`, `.deal-btn`.
- *Secondary / ghost* — прозрачный фон, `--border-default`, `--text-secondary`. Reset, Back, Export.
- *Danger* — `--wrong` обводка на hover. Reset Progress.
- *Action* (спот-действия) — фон из `--c-<action>`, текст из `--c-<action>-fg`. Open/Call/3bet/4bet. **Fold-кнопки нет** (решение P-014, 2026-05-30): рука без действия = implicit fold (полный и частичный), явная кнопка редундантна. В редакторе панель Action = сетка 2×2 (Open/3-Bet/Call/4-Bet) + широкая кнопка-корзина 🗑 (Clear) снизу. Цвет fold (`#1e2a22`) остаётся только для отрисовки незалитого остатка в ячейках матрицы.
- Размеры: sm (`--sp-2 --sp-3`, `--fs-sm`), md (`--sp-3 --sp-4`, `--fs-base`).

**Игральные карты** (герой, GG-богатство). Анатомия: белое лицо, `--radius-md`, `--shadow-card`, ранг в углу + крупная масть в центре. **4-цветная масть** (решение проекта): червы `#ae0d03`, бубны `#2235c5`, трефы `#33ae2b`, пики `#504f4f`. Ассеты — CC0 SVG-колода `saulspatz/SVGCards` (vertical, jumbo-index), перекрашена и лежит в `cards/` (см. roadmap B.1-cards). Цель Track B: добавить лёгкий градиент на лицо, глянец, аккуратную рубашку для card-back. Сейчас (CSS-карты hero): `.card`, `.card-front`, `.card-back`, `.card-corner`, `.card-center` — их предстоит заменить на SVG-ассеты.

**Решение по типу карты (2026-05-29).** Рассматривали плоскую двухтоновую плитку в стиле GTO Wizard (вся карта залита цветом масти, ранг буквой) как альтернативу. **Отклонено** — берём реалистичную перекрашенную колоду. Причина: плитка — это GTOW-дисциплина, а карты у нас «герой» (§1/§6), плюс её главный плюс (читаемость на крошечном размере) у нас не стреляет, т.к. оппоненты сидят рубашками. Не переоткрывать без явной причины.

**Где показываются вскрытые лица (2026-05-29).** Face-up лица — только у **hero** (Drill/Learn) и на **борде** (постфлоп). Оппоненты в Drill/Learn — всегда рубашки (`.mini-back`), их лиц не видно. Вскрытые лица оппонентов появляются **только в анализаторе/реплеере/парсере на шоудауне** (территория Milestone 3). Это определяет, где вообще нужен рендер SVG-лиц.

**UI-карточки / панели** (дисциплина GTOW). `--surface-1` или `--surface-2`, `--border-subtle`, `--radius-lg`, `--sp-5` padding. Сейчас: `.learn-entry-card`, `.panel`.

**Сегментированные контролы.** Группа кнопок-переключателей (spot picker, scope picker, game-type). Активная — `--accent` обводка + `--accent-bg`. Сейчас: `.seg-buttons`, `.seg`.

**Табы.** Верхний навбар (`.mode-tab`) — активный с золотой рамкой. Правые табы (`.right-tab`). На мобилке (Track B.4) верхние табы переедут вниз.

**Stat-плашки / счётчики.** `--surface-2` фон, крупное число (`--fs-lg`, `--font-ui`, `--fw-bold`), мелкий ярлык под ним (`--fs-xs`, `--tracking-wide`, `--text-tertiary`). Сейчас: `.learn-counter`, `.lc-num`, `.lc-label`.

**Стол** (герой). Форма — **стадион/racetrack** (прямые длинные борта + закруглённые торцы), соотношение ~2:1 (680×330 на десктопе), НЕ круг/эллипс. Реализация: `border-radius:9999px` (CSS сам обрезает до половины высоты → чистый стадион при любых размерах бокса, удобно для мобильной адаптации). Координаты рассадки 6/8/9-max сверены с GG-референсами и подтверждены. `.table-wrap`, `.table-rim` (борт, `--shadow-table`), `.table-felt` (градиентный фелт), `.seats` + `.seat-circle` (позиции), `.bet-chip` (фишки), `.pot-block`.

**Место игрока (seat).** Без аватарки. **Универсальная анатомия (одинаково для всех мест):** карты ВСЕГДА сверху, стек ВСЕГДА снизу. Сверху вниз: рубашки карт (`.seat-cards` — две `.mini-back` 28×39, тукаются ЗА кружок через `.seat-avatar`, z-index ниже кружка → торчат над аватаром, индикатор «игрок держит карты») → позиция-токен (`.seat-circle`, кружок с названием позиции = «ник»; hero — золотая обводка, villain — красная, BTN — бейдж `D`) → метка `.seat-name` ТОЛЬКО у hero («Hero»; у оппонентов нет — позиция уже в кружке) → стек (`.seat-stack`, BB — светлый текст 12px/600 на тёмной полупрозрачной пилюле, чтобы читался на ярком фелте и перекрывал betting line за собой; B.1-polish 2026-05-29). У hero вместо рубашек — реальные face-up карты в отдельном `.hero-cards` под столом. Состояние **fold** (`.seat.is-folded`): рубашки скрыты, всё место `opacity:.34 + grayscale(.6)`. Рубашка `.mini-back` — **синяя** (`linear-gradient(135deg,#1d3a6e,#0a1428)`, согласована с card-back из §3) со светло-синей внутренней рамкой. Самые верхние места в `SLOT_POS` опущены до 16–22%, чтобы карты над ними не вылезали за стол в заголовок спота. Прототип-эталон: `static/table_preview.html` (опережает финал — там ещё была направленная раскладка).

**Фишки / банк** (герой, GG-богатство; B.1-legacy, done 2026-05-30). Реалистичные казино-фишки **с номиналами**, чистый CSS без ассетов. Кэш-модель, всё в BB. Номиналы `0.1 / 0.5 / 1 / 5 / 25`; **цвет = номинал** (якоря: белый=1, красный=5, зелёный=25; стальной 0.1, голубой 0.5). Все диски **одного диаметра** — как в реальном казино, номинал читается цветом, не размером. Анатомия диска: top-down эллипс 19×8, насечки-«споты» по ободу (`repeating-conic-gradient`, стаггер `--rot` по фишкам), тёмный нижний шов + верхний блик → читается как пачка отдельных дисков; светлый инлей с мелкой цифрой номинала на верхнем диске. **Ставка** = одна смешанная стопка (крупные внизу) + пилюля точной суммы (`.chip-stack`/`.coins`/`.coin`/`.chip-amt`). **Банк** = те же фишки сгруппированы по номиналам в отдельные стопки бок о бок под текстом «POT … BB» (`.chip-pot`/`.pot-row`). Палитры `--body/--edge/--inlay/--ink/--ring` в классах `.d-01..d-25`. Хелперы рендера — общий модуль **`static/chips.js`** (`chipMakeChange` жадный размен с округлением до 0.1, `chipBetStack`, `chipPotStacks`; единый источник для Drill и Postflop, грузится первым как `cards.js`). Реф — KO Poker / PokerStars (стопочки в центре, виден размер пота). Прототип-эталон: `static/chips_preview.html`.

**Куда что рисуется (важно — иначе задвоение).** Деньги «перед игроком» (ставка-стопка) vs «в банке» (пот-стопки) — разные состояния, не дублировать. **Префлоп (Drill):** блайнды/open/3bet стоят у позиций по `SLOT_CHIP`; центрального пота фишками НЕТ (он бы дублировал блайнды — пот 1.5 = SB 0.5 + BB 1, уже стоящие у мест), размер банка несёт текст `.pot-block`. **Постфлоп:** ставки прошлых улиц собраны в центр → там `chipPotStacks`. Ставка виллана текущей улицы — `chipBetStack` (Postflop уже на новой системе). Это исправление от 2026-05-30; деталь — `problems.md`/`roadmap.md` B.1-legacy.

**Решение по фишкам (2026-05-30).** Рассматривали: (а) плоские токены с action-цветом (синий open, красный 3bet) — отклонено, цвет теперь несёт номинал, тип ставки читается по позиции и сумме (реалистичнее); (б) одну фишку с числом vs стопку — выбрана стопка; (в) ряд `1/5/10/50` — заменён на казино-ряд `0.1/0.5/1/5/25` (схема 1-5-25 даёт минимум фишек при размене, +0.1/0.5 для блайндов и дробных опенов). Диаметр уменьшен (с «тарелок» 28×10 до 19×8). **Старые `.bet-chip`/`.chip-*` оставлены** в CSS для замороженного postflop — при разморозке перевести на эту систему.

**Матрица 13×13** (дисциплина GTOW). Плотная сетка, каждая клетка кодирована частотой действия через `--c-<action>`. Цель Track B.3: выраженные градиенты для mixed-частот, hover-tooltip с точной частотой. Сейчас: `.editor-matrix`, `.viz` матрицы.

**Feedback-strip.** Тонкая полоса вердикта после ответа. `--surface-2`, вердикт `--correct`/`--wrong`. Сейчас: `.learn-feedback-strip`, `.lfb-verdict`.

## 4. Адаптив / breakpoints (Track B.4)

```
/* mobile-first логика, но текущий код desktop-first — переход постепенный */
--bp-mobile:  480px    /* телефоны */
--bp-tablet:  768px    /* планшеты, граница single→multi-column */
--bp-desktop: 1024px   /* полный layout */
```
Ключевые правила (детали — в `roadmap.md` Track B.4):
- `< 768px`: `.layout` из 3-колоночного grid → single column, sidebars в off-canvas drawer.
- Стол: фикс. 560px → `min(92vw, 560px)` с пропорциональным масштабом.
- Верхний навбар табов → нижний (bottom nav), tap-таргеты ≥ 44px.
- Матрица: на весь экран + выбор спота отдельной панелью снизу.

| magic font-sizes | `--fs-*` | заменить |
| инлайн-тени | `--shadow-*` | вынести в токены |

**Порядок применения** (когда дойдём до кода): сначала добавить новые токены в `:root` НЕ удаляя старые (старые → алиасы на новые), потом постепенно переводить компоненты, в конце удалить старые алиасы. Так ничего не ломается в процессе.

## 6. Что НЕ делаем

- Не вводим CSS-фреймворк (Tailwind и т.п.) — у нас чистый CSS без сборщика, это сознательное решение из CLAUDE.md.
- Не привлекаем человека-дизайнера на этапе беты — система выше + GG/GTOW-референсы достаточны. Дизайнер окупится на переходе бета → опен, тогда он уточнит токены в Figma (Dev Mode), а не перерисует всё с нуля.
- Не трогаем стол/карты в сторону «строже» — они герой, им богатство.
