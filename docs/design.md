# Poker Trainer — Design System

Создан: 2026-05-28. Источник истины по визуалу. Этот документ описывает **целевую** дизайн-систему; код (`static/style.css`) подтягивается к ней постепенно в рамках Track B (см. `roadmap.md`). Пока документ опережает код — это нормально, он задаёт направление.

---

## 1. Направление

Сочетание двух референсов:

- **GGPoker — богатство «героя».** Стол и карты выглядят премиально и тактильно: насыщенный фелт с глубиной, выраженный деревянный rim, карты с чистой типографикой, мягкими тенями и аккуратной рубашкой, фишки с denominations. Это эмоциональный центр экрана.
- **GTO Wizard — строгость всего остального.** Интерфейс вокруг стола — дисциплинированный, сетка-выверенный, информационно-чистый. Приглушённая профессиональная палитра, один уверенный акцент (золото) используется скупо, резкая типографика, плотная читаемая матрица 13×13. Энергетика «серьёзный инструмент для изучения», не казино.

Правило компоновки: **роскошь концентрируется в столе и картах, всё остальное — сдержанно и строго.** Если сомневаешься, делать элемент богаче или строже — делай строже, кроме случаев когда это стол/карты/фишки.

## 3. Инвентарь компонентов

Каждый компонент — его назначение, анатомия, какие токены берёт. Цель: чтобы при редизайне (и при работе будущего дизайнера) был единый список «что вообще есть в системе».

**Кнопки.**
- *Primary* — золотая обводка + `--accent-bg` фон + `--accent` текст. Один primary на экран (Next, Учить, Save). Сейчас: `.learn-start-btn`, `.learn-next-btn`, `.deal-btn`.
- *Secondary / ghost* — прозрачный фон, `--border-default`, `--text-secondary`. Reset, Back, Export.
- *Danger* — `--wrong` обводка на hover. Reset Progress.
- *Action* (спот-действия) — фон из `--c-<action>`, текст из `--c-<action>-fg`. Open/Call/3bet/4bet. **Fold-кнопки нет** (решение P-014, 2026-05-30): рука без действия = implicit fold (полный и частичный), явная кнопка редундантна. В редакторе панель Action = сетка 2×2 (Open/3-Bet/Call/4-Bet) + широкая кнопка-корзина 🗑 (Clear) снизу. Цвет fold (`#1e2a22`) остаётся только для отрисовки незалитого остатка в ячейках матрицы.
- Размеры: sm (`--sp-2 --sp-3`, `--fs-sm`), md (`--sp-3 --sp-4`, `--fs-base`).

**Игральные карты** (герой, GG-богатство). Анатомия (обновлено 2026-06-14 → **GTOW-плитка**, см. «Решение по типу карты» ниже): плоская плитка цвета масти, `--radius-md`, огромная масть-вотермарка (opacity .26) с заходом за край (clip по форме карты), крупный белый ранг по центру, **без угловых индексов**; десятка = «T». **4-цветная плитка** (углублено под белый текст): червы `#b01217`, бубны `#1f3bb0`, трефы `#2a8a36`, пики `#3f3f42`. Источник — детерминированный генератор `cards/gen_cards.py` (52 SVG в `cards/`, имена прежние → drop-in). Рендер hero — `.card.card-svg` (`<img src=/cards/…svg>`); прежний CSS-рендер (`.card-front`/`.card-corner`/`.card-center`) остаётся фоллбэком когда `cardSvgUrl→null`. Прежняя реалистичная колода (saulspatz) — в бэкапе `cards/_backup_realistic_20260614/`. **Рубашка `.card-back` — done (2026-06-03):** собственный ассет `cards/goldBack.svg` (чёрный+золото, ornate), см. ниже про seat и roadmap B.1-cards. **Веер hero — УБРАН (2026-06-14, по решению пользователя):** hero-карты снова вертикальны (базовый `.hero-cards`: flex + gap, без наклона/нахлёста/pivot). Токен `--card-fan: 9deg` оставлен — теперь питает только subtle-веер рубашек оппонентов (`.mini-back`); `#heroCards`/`#learnHeroCards` его больше не используют, `@keyframes deal` для hero даёт rotate(0). Раньше (2026-05-31…06-14) hero-карты фанились ±9° (pivot снизу + нахлёст −14px). **Зазор hero-рук сжат до 2px (2026-06-15, по решению пользователя)** — карты стоят почти вплотную. Правило **scoped** на `#heroCards`/`#learnHeroCards` (Drill + Learn); базовый `.hero-cards` остаётся `gap:12px` — его переиспользует постфлоп-борд (`#pfBoard`), где community-карты должны сохранять зазор. Мобильный оверрайд тоже 2px (был 8px).

**Решение по типу карты (2026-05-29 → ПЕРЕСМОТРЕНО 2026-06-14).** Сначала плитку GTO Wizard отклонили в пользу реалистичной колоды. **2026-06-14 по решению пользователя развернули обратно на плитку** — в «чистом-крупном» прочтении: цвет масти + огромная масть-вотермарка с заходом за край + крупный ранг по центру, без углов, аккуратные векторные масти (острый кончик червы, тонкая пика, чёткий трилистник трефы). Это примиряет «карта-герой» с GTOW-раскладкой: раскладка строгая и читаемая, исполнение крупное и тактильное. Десятка = «T». Прежняя реалистичная колода — в бэкапе `cards/_backup_realistic_20260614/`. Детали и обоснование — CLAUDE.md решение #12 (поправка 2026-06-14).

**Где показываются вскрытые лица (2026-05-29).** Face-up лица — только у **hero** (Drill/Learn) и на **борде** (постфлоп). Оппоненты в Drill/Learn — всегда рубашки (`.mini-back`), их лиц не видно. Вскрытые лица оппонентов появляются **только в анализаторе/реплеере/парсере на шоудауне** (территория Milestone 3). Это определяет, где вообще нужен рендер SVG-лиц.

**UI-карточки / панели** (дисциплина GTOW). `--surface-1` или `--surface-2`, `--border-subtle`, `--radius-lg`, `--sp-5` padding. Сейчас: `.learn-entry-card`, `.panel`.

**Сегментированные контролы.** Группа кнопок-переключателей (spot picker, scope picker, game-type). Активная — `--accent` обводка + `--accent-bg`. Сейчас: `.seg-buttons`, `.seg`.

**Табы.** Верхний навбар (`.mode-tab`) — активный с золотой рамкой. Правые табы (`.right-tab`). На мобилке (Track B.4) верхние табы переедут вниз.

**Stat-плашки / счётчики.** `--surface-2` фон, крупное число (`--fs-lg`, `--font-ui`, `--fw-bold`), мелкий ярлык под ним (`--fs-xs`, `--tracking-wide`, `--text-tertiary`). Сейчас: `.learn-counter`, `.lc-num`, `.lc-label`.

**Стол** (герой). Форма — **стадион/racetrack** (прямые длинные борта + закруглённые торцы), соотношение ~2:1 (680×330 на десктопе), НЕ круг/эллипс. Реализация: `border-radius:9999px` (CSS сам обрезает до половины высоты → чистый стадион при любых размерах бокса, удобно для мобильной адаптации). Координаты рассадки 6/8/9-max сверены с GG-референсами и подтверждены. `.table-wrap`, `.table-rim` (борт, `--shadow-table`), `.table-felt` (градиентный фелт), `.seats` + `.seat-circle` (позиции), `.bet-chip` (фишки), `.pot-block`.

**Место игрока (seat).** Без аватарки. **Универсальная анатомия (одинаково для всех мест):** карты ВСЕГДА сверху, стек ВСЕГДА снизу. Сверху вниз: рубашки карт (`.seat-cards` — две `.mini-back` 28×39, тукаются ЗА кружок через `.seat-avatar`, z-index ниже кружка → торчат над аватаром, индикатор «игрок держит карты») → позиция-токен (`.seat-circle`, кружок с названием позиции = «ник»; hero — золотая обводка, villain — красная, BTN — бейдж `D`) → метка `.seat-name` ТОЛЬКО у hero («Hero»; у оппонентов нет — позиция уже в кружке) → стек (`.seat-stack`, BB — светлый текст 12px/600 на тёмной полупрозрачной пилюле, чтобы читался на ярком фелте и перекрывал betting line за собой; B.1-polish 2026-05-29). У hero вместо рубашек — реальные face-up карты в отдельном `.hero-cards` под столом. Состояние **fold** (`.seat.is-folded`): рубашки скрыты, всё место `opacity:.34 + grayscale(.6)`. Рубашка `.mini-back` — **чёрный+золото, ornate** (ассет `cards/goldBack.svg`: двойная золотая рамка, гильош-решётка, медальон со спейдом; единый с `.card-back`, 2026-06-03). Раньше была синим CSS-градиентом. Самые верхние места в `SLOT_POS` опущены до 16–22%, чтобы карты над ними не вылезали за стол в заголовок спота. Прототип-эталон: `static/table_preview.html` (опережает финал — там ещё была направленная раскладка).

**Фишки / банк** (герой, GG-богатство; B.1-legacy, done 2026-05-30). Реалистичные казино-фишки **с номиналами**, чистый CSS без ассетов. Кэш-модель, всё в BB. Номиналы `0.1 / 0.5 / 1 / 5 / 25`; **цвет = номинал** (якоря: белый=1, красный=5, зелёный=25; стальной 0.1, голубой 0.5). Все диски **одного диаметра** — как в реальном казино, номинал читается цветом, не размером. Анатомия диска: top-down эллипс 19×8, насечки-«споты» по ободу (`repeating-conic-gradient`, стаггер `--rot` по фишкам), тёмный нижний шов + верхний блик → читается как пачка отдельных дисков; светлый инлей с мелкой цифрой номинала на верхнем диске. **Ставка** = одна смешанная стопка (крупные внизу) + пилюля точной суммы (`.chip-stack`/`.coins`/`.coin`/`.chip-amt`). **Банк** = те же фишки сгруппированы по номиналам в отдельные стопки бок о бок под текстом «POT … BB» (`.chip-pot`/`.pot-row`). Палитры `--body/--edge/--inlay/--ink/--ring` в классах `.d-01..d-25`. Хелперы рендера — общий модуль **`static/chips.js`** (`chipMakeChange` жадный размен с округлением до 0.1, `chipBetStack`, `chipPotStacks`; единый источник для Drill и Postflop, грузится первым как `cards.js`). **Масштаб (done 2026-05-31, B.1-gg-ref):** размер фишки ∝ ширине стола по GG/PokerLay-методике (§7) — `chipScaleFor(W)=clamp(W/680,0.6,1)`; билдеры принимают `scale`, выставляют `--cs` на обёртке, CSS считает диск/шрифт через `calc(...*var(--cs,1))`. Мобильный костыль `scale(.82)` убран (стал частным случаем). Drill передаёт scale от живого стола; postflop на scale 1 (заморожен). Реф — KO Poker / PokerStars (стопочки в центре, виден размер пота). Прототип-эталон: `static/chips_preview.html`.

**Куда что рисуется (важно — иначе задвоение).** Деньги «перед игроком» (ставка-стопка) vs «в банке» (пот-стопки) — разные состояния, не дублировать. **Префлоп (Drill):** блайнды/open/3bet стоят у позиций по `SLOT_CHIP`; центрального пота фишками НЕТ (он бы дублировал блайнды — пот 1.5 = SB 0.5 + BB 1, уже стоящие у мест), размер банка несёт текст `.pot-block`. **Постфлоп:** ставки прошлых улиц собраны в центр → там `chipPotStacks`. ## 7. GG-стол — числовой референс (из PokerLay overlay, 2026-05-31)

Извлечено из стороннего оверлея **PokerLay** (Electron-приложение, рисует ClubGG-скин поверх живого покер-клиента). Архив прислал пользователь как «файлы для открытого использования».

**Что отсюда берём и что НЕ берём.** Самих картинок (фелт, карты, фишки, кнопки) в архиве нет — скин подгружался по абсолютным путям с машины автора (`C:/Users/User/NormalPoker/skins/clubgg/...`) и это **проприетарная графика GG/ClubGG**. Её не копируем. Берём только **числа**: нормализованные координаты рассадки, формулу масштаба фишек, геометрию веера, точные цвета пота. Пропорции и цвета авторским правом не охраняются — это безопасный референс «как у GG устроены метрики стола», чтобы сверять наши значения. **Не цель** — пиксельно повторять PokerLay; это сверочная таблица, а наш стол остаётся своим (стадион 2:1, §3).

**Рассадка (нормализованные доли окна).** Формат строки в их конфиге — `flag cx cy R rot` (точная семантика полей в архиве не задокументирована; значения — это данные, толкование полей — наше предположение по поведению). У GG два пресета:

```
6/7/8/9-max:   1  0.635  0.590  0.545  0     ← общий «ring» пресет, единый центр+радиус
2/3/4/5/10-max: 0  0.637  0.719  0.660  0
```

Читать как: центр окружности рассадки ≈ (0.635, 0.590) от размеров окна, радиус ≈ 0.545. Плюс пер-сит микро-сдвиги (`seat_off_*`) в пикселях для ручной подгонки отдельных мест. **Сверка:** наш §3 говорит «координаты 6/8/9-max сверены с GG и подтверждены» — этот пресет можно использовать как независимую вторую проверку, если будем пересчитывать рассадку.

**Масштаб фишки ставки** (их «закон GG», документирован прямо в архиве):

```
размер_фишки_px = chip_px * scale,   где scale = min(W/chip_ref_w, H/chip_ref_h), но не ниже chip_px_min
референс-окно:  chip_ref_w = 1080,  chip_ref_h = 1920   (портрет — базис GG)
база размера:   chip_px = 41 (при scale=1),  chip_px_min = 4 (пол)
шрифт суммы:    chip_bb_font = 28 (база),  chip_bb_min = 6 (пол),  та же формула
```

Практически для горизонтального окна высотой H: `фишка_px ≈ 41 * H / 1920`. Полезно как принцип: GG привязывает ВСЕ размеры к одному референс-окну 1080×1920 и масштабирует от меньшей стороны. У нас фишки сейчас фиксированы — если дойдём до пропорционального масштаба стола (Track B.4 / мобилка), эта формула — готовый паттерн.

**Веер карт / геометрия (для постфлоп-борда и hero-руки).**

```
fan        : 10 6 0.66 1.0 2 0.645 0.160 0   ← угол ≈10°, spread/curve параметры
card_size  : 0.15        ← доля размера карты от окна
cards_down : 6 0.90 0    ← рубашки оппонентов
own_hand_ang : 1.04   own_hand_gap : 1.22   own_hand_sz : 1.00   ← наклон/зазор/размер карт hero
```

**Цвета пота (точные значения GG-пилюли).**

```
pot_colors : #c2c8c2 (лейбл, холодный серый)   #ffd633 (сумма, золото)
pot_pill   : #cc16331f   ← подложка пилюли, 8-значный hex
```

`#ffd633` — более «жёлтое» золото суммы, чем наш `--accent #c9a84c`; стоит иметь в виду как альтернативный оттенок для числа в банке. `pot_pill #cc16331f` неоднозначен по порядку байт: как CSS `RRGGBBAA` это полупрозрачный малиновый `rgba(204,22,51,.12)`, как `AARRGGBB` — тёмно-зелёный `rgba(22,51,31,.8)`; второе вероятнее для подложки на фелте, но точно не проверить без бинарника. Наш текущий pot pill (§3, тёмный + золотая обводка) этому не противоречит — берём только как данные о том, что GG держит пот холодно-серым с золотой суммой.

**Где лежит распакованный архив** (если понадобится дозабрать числа): рабочая распаковка делалась в сессии, в репозиторий ничего из PokerLay не клали (и не кладём — там чужой бинарник 189 МБ и проприетарный скин). Все нужные значения — выше.

**Прозрачный surround (2026-06-15, по решению пользователя).** AI-ассет приходил RGB с запечённым **чёрным** прямоугольным фоном вокруг стола. На фоне страницы `--bg #080d0a` (тёмный charcoal-green, не чистый чёрный) края PNG читались как видимый чёрный прямоугольник — некрасиво. Фикс: чёрный surround вырезан в alpha=0 (flood-fill от четырёх углов по `max(R,G,B) ≤ 20`, лёгкий feather 0.6px по кромке; рейл `≈56+` намного выше порога → срез чистый по краю рейла, фелт/рейл целы). Теперь стол «плавает» на фоне страницы и блендится при ЛЮБОМ фоне (выбран этот путь, а не подгонка `--bg` под чёрный — так сохраняется дизайн-токен `--surface-0` и не привязываемся к чёрному). Запечённая мягкая тень (была почти-чёрная) ушла вместе с surround → восполнена CSS: `.table-bg { filter: drop-shadow(0 8px 28px rgba(0,0,0,.55)) }` (тень следует за alpha-формой стадиона, прямоугольника не даёт). Имя файла прежнее (`table.png`, теперь **RGBA**) → drop-in, CSS-геометрия не менялась; cache-bust `?v=20260615a` на 3 `<img class="table-bg">` в `index.html` + `style.css?v=20260615a`. Бэкап непрозрачного оригинала — `static/table_art/table_opaque_20260615.png` (откат: скопировать обратно в `table.png`). Конвертацию делал ad-hoc PIL-скрипт (flood-fill); `render_table.py` УЖЕ выдаёт прозрачный RGBA, так что при возврате к процедурному рендеру отдельный шаг не нужен.

**Пересмотр §6:** «не вводим бинарные ассеты / не привлекаем дизайнера до беты» — отменён для арт-героя. Реализм через растровые ассеты — основной путь стола/карт; GTOW-дисциплина интерфейса остаётся на CSS.
