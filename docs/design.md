# Poker Trainer — Design System

Создан: 2026-05-28. Источник истины по визуалу. Этот документ описывает **целевую** дизайн-систему; код (`static/style.css`) подтягивается к ней постепенно в рамках Track B (см. `roadmap.md`). Пока документ опережает код — это нормально, он задаёт направление.

---

## 1. Направление

Сочетание двух референсов:

- **GGPoker — богатство «героя».** Стол и карты выглядят премиально и тактильно: насыщенный фелт с глубиной, выраженный деревянный rim, карты с чистой типографикой, мягкими тенями и аккуратной рубашкой, фишки с denominations. Это эмоциональный центр экрана.
- **GTO Wizard — строгость всего остального.** Интерфейс вокруг стола — дисциплинированный, сетка-выверенный, информационно-чистый. Приглушённая профессиональная палитра, один уверенный акцент (золото) используется скупо, резкая типографика, плотная читаемая матрица 13×13. Энергетика «серьёзный инструмент для изучения», не казино.

Правило компоновки: **роскошь концентрируется в столе и картах, всё остальное — сдержанно и строго.** Если сомневаешься, делать элемент богаче или строже — делай строже, кроме случаев когда это стол/карты/фишки.

## 2. Токены

Это целевой набор. Текущий `:root` имеет ~60% (см. §5 маппинг). Жирным — то, чего сейчас нет и надо добавить.

### 2.1. Палитра

**Нейтрали — холодный графит (chrome-redesign §1, ПРИМЕНЕНО в живой код 2026-06-18).**
Ушли от прежнего charcoal-green подтона (зелёный отлив панелей выглядел грязно) к нейтральному графиту. Матрица остаётся зелёной и теперь «выстреливает» на нейтральной обвязке. Золото-акцент и зелёный фелт/экшен-цвета не тронуты.
```
--surface-0 / --bg:  #0a0b0e   /* app background (было #080d0a green) */
--surface-1 / --panel: #14161b /* панели, sidebar (было #0d1510 green) */
--surface-2:  #1c1f26   /* приподнятые: карточки, stat-плашки, аватары */
--surface-3:  #21242b   /* модалки, popover, hint-overlay, tooltip */
--recessed:   #0e0f13   /* утопленные дорожки: слайдеры, wells */
--stroke:     #2f333b   /* нейтральный хайрлайн (было зелёное #2a3a2a) */

--border-subtle:  rgba(255,255,255,.06)   /* = текущий --panel-border */
--border-default: rgba(255,255,255,.10)                                       ⟵ НОВОЕ
--border-strong:  rgba(255,255,255,.16)                                       ⟵ НОВОЕ
```
Живой `:root` сейчас держит: `--bg`, `--panel`, `--surface-2`, `--surface-3`, `--recessed`, `--stroke`. Зелёные хардкоды обвязки (тумблеры, слайдеры, скроллбар, поповеры, аватар, пустая клетка матрицы) переведены на эти токены.

**Текстовая шкала (графит, chrome-redesign §1).**
```
--text-primary:   #e6e8ec   /* = текущий --text     (было #dceae1 green) */
--text-secondary: #aab0bc   /* = текущий --text-mid  (было #9ab8a4 green) */
--text-tertiary:  #79808c   /* = текущий --text-dim  (было #5a7a65 green) */
--text-disabled:  rgba(220,234,225,.32)                                       ⟵ НОВОЕ
```

**Акцент (золото — скупо, по принципу GTOW «один уверенный акцент»).**
```
--accent:     #c9a84c   /* = текущий --gold */
--accent-hi:  #e8c97a   /* = текущий --gold-hi */
--accent-bg:  rgba(201,168,76,.12)   /* = текущий --gold-dim, чуть прозрачнее */
```
Применение: активный таб, primary-кнопка, выделение «твоя позиция» за столом. **Не** красить золотом всё подряд — это убивает иерархию.

**Фелт и стол (богатство GG — яркий казино-зелёный + чёрный рейл, по референсу CoinPoker, B.1-polish 2026-05-29).**
```
--felt:        #1d6b40   /* база, vivid casino green */
--felt-hi:     #379a59   /* светлый центр фелта */
--felt-shadow: #0b3a21   /* затемнение к рейлу */
--felt-line:   rgba(150,225,180,.42)  /* betting line — светлый кант у края */
--rim:         #161616   /* чёрный мягкий рейл (был деревянный) */
--rim-hi:      #353535   /* блик сверху рейла */
```
Эти — «герой», их не трогаем в сторону приглушения. **Реализация (готово):** фелт = радиальный градиент (светлый центр → тёмный край) + vignette + тёмный шов; рейл = вертикальный градиент (блик сверху → чёрный снизу) + внешняя drop-тень + inset блик/тень для объёма; betting line = `.table-felt::before` (светлый кант по внутр. краю). **Решение по цвету (2026-05-29):** ушли от деревянного коричневого борта к чёрному и от приглушённого зелёного к яркому — по референсу реального казино-стола. Дерево/бордо/тил/графит рассматривались, выбран яркий зелёный + чёрный рейл.

**Семантика.**
```

--wrong:       #c0392b   --wrong-bg:   rgba(192,57,43,.12)
```

## 3. Инвентарь компонентов

Каждый компонент — его назначение, анатомия, какие токены берёт. Цель: чтобы при редизайне (и при работе будущего дизайнера) был единый список «что вообще есть в системе».

**Кнопки.**
- *Primary* — золотая обводка + `--accent-bg` фон + `--accent` текст. Один primary на экран (Next, Учить, Save). Сейчас: `.learn-start-btn`, `.learn-next-btn`, `.deal-btn`.
- *Secondary / ghost* — прозрачный фон, `--border-default`, `--text-secondary`. Reset, Back, Export.
- *Danger* — `--wrong` обводка на hover. Reset Progress.
- *Action* (спот-действия) — фон из `--c-<action>`, текст из `--c-<action>-fg`. Open/Call/3bet/4bet. **Fold-кнопки нет** (решение P-014, 2026-05-30): рука без действия = implicit fold (полный и частичный), явная кнопка редундантна. В редакторе панель Action = сетка 2×2 (Open/3-Bet/Call/4-Bet) + широкая кнопка-корзина 🗑 (Clear) снизу. Цвет fold (`#1e2a22`) остаётся только для отрисовки незалитого остатка в ячейках матрицы.
- Размеры: sm (`--sp-2 --sp-3`, `--fs-sm`), md (`--sp-3 --sp-4`, `--fs-base`).

Сменила LayIpoker-колоду (бэкап `cards/_backup_layi_20260623/`). См. CLAUDE.md #12 поправка 2026-06-23.** Анатомия (обновлено 2026-06-14 → **GTOW-плитка**, см. «Решение по типу карты» ниже): плоская плитка цвета масти, `--radius-md`, огромная масть-вотермарка (opacity .26) с заходом за край (clip по форме карты), крупный белый ранг по центру, **без угловых индексов**; десятка = «T». **4-цветная плитка** (углублено под белый текст): червы `#b01217`, бубны `#1f3bb0`, трефы `#2a8a36`, пики `#3f3f42`. **Стол** (герой). Форма — **стадион/racetrack** (прямые длинные борта + закруглённые торцы), соотношение ~2:1 (680×330 на десктопе), НЕ круг/эллипс. Прототип-эталон: `static/table_preview.html` (опережает финал — там ещё была направленная раскладка).

**Фишки / банк** (герой, GG-богатство; B.1-legacy 2026-05-30 → casino-3D рендер B.1-chips-3d 2026-06-15). Реалистичные казино-фишки **с номиналами**, чистый CSS без ассетов. Кэш-модель, всё в BB. Номиналы `0.1 / 0.5 / 1 / 5 / 25`; **цвет = номинал** (якоря: белый=1, красный=5, зелёный=25; стальной 0.1, голубой 0.5). ## 7. GG-стол — числовой референс (из PokerLay overlay, 2026-05-31)

Извлечено из стороннего оверлея **PokerLay** (Electron-приложение, рисует ClubGG-скин поверх живого покер-клиента). Архив прислал пользователь как «файлы для открытого использования».

**Что отсюда берём и что НЕ берём.** Самих картинок (фелт, карты, фишки, кнопки) в архиве нет — скин подгружался по абсолютным путям с машины автора (`C:/Users/User/NormalPoker/skins/clubgg/...`) и это **проприетарная графика GG/ClubGG**. Её не копируем. Берём только **числа**: нормализованные координаты рассадки, формулу масштаба фишек, геометрию веера, точные цвета пота. Пропорции и цвета авторским правом не охраняются — это безопасный референс «как у GG устроены метрики стола», чтобы сверять наши значения. **Не цель** — пиксельно повторять PokerLay; это сверочная таблица, а наш стол остаётся своим (стадион 2:1, §3).

**Рассадка (нормализованные доли окна).** Формат строки в их конфиге — `flag cx cy R rot` (точная семантика полей в архиве не задокументирована; значения — это данные, толкование полей — наше предположение по поведению). У GG два пресета:

```
6/7/8/9-max:   1  0.635  0.590  0.545  0     ← общий «ring» пресет, единый центр+радиус
2/3/4/5/10-max: 0  0.637  0.719  0.660  0
```

Читать как: центр окружности рассадки ≈ (0.635, 0.590) от размеров окна, радиус ≈ 0.545. Плюс пер-сит микро-сдвиги (`seat_off_*`) в пикселях для ручной подгонки отдельных мест. **Сверка:** наш §3 говорит «координаты 6/8/9-max сверены с GG и подтверждены» — этот пресет можно использовать как независимую вторую проверку, если будем пересчитывать рассадку.

**Масштаб фишки ставки** (их «закон GG», документирован прямо в архиве):

```
размер_фишки_px = chip_px * scale,   где scale = min(W/chip_ref_w, H/chip_ref_h), но не ниже chip_px_min
референс-окно:  chip_ref_w = 1080,  chip_ref_h = 1920   (портрет — базис GG)
база размера:   chip_px = 41 (при scale=1),  chip_px_min = 4 (пол)
шрифт суммы:    chip_bb_font = 28 (база),  chip_bb_min = 6 (пол),  та же формула
```

Практически для горизонтального окна высотой H: `фишка_px ≈ 41 * H / 1920`. Полезно как принцип: GG привязывает ВСЕ размеры к одному референс-окну 1080×1920 и масштабирует от меньшей стороны. У нас фишки сейчас фиксированы — если дойдём до пропорционального масштаба стола (Track B.4 / мобилка), эта формула — готовый паттерн.

**Веер карт / геометрия (для постфлоп-борда и hero-руки).**

```
fan        : 10 6 0.66 1.0 2 0.645 0.160 0   ← угол ≈10°, spread/curve параметры
card_size  : 0.15        ← доля размера карты от окна
cards_down : 6 0.90 0    ← рубашки оппонентов
own_hand_ang : 1.04   own_hand_gap : 1.22   own_hand_sz : 1.00   ← наклон/зазор/размер карт hero
```

**Цвета пота (точные значения GG-пилюли).**

```
pot_colors : #c2c8c2 (лейбл, холодный серый)   #ffd633 (сумма, золото)
pot_pill   : #cc16331f   ← подложка пилюли, 8-значный hex
```

`#ffd633` — более «жёлтое» золото суммы, чем наш `--accent #c9a84c`; стоит иметь в виду как альтернативный оттенок для числа в банке. `pot_pill #cc16331f` неоднозначен по порядку байт: как CSS `RRGGBBAA` это полупрозрачный малиновый `rgba(204,22,51,.12)`, как `AARRGGBB` — тёмно-зелёный `rgba(22,51,31,.8)`; второе вероятнее для подложки на фелте, но точно не проверить без бинарника. Наш текущий pot pill (§3, тёмный + золотая обводка) этому не противоречит — берём только как данные о том, что GG держит пот холодно-серым с золотой суммой.
