### A.1 — History-логирование в Card *(в работе)*
Сейчас в `srs_state/*.srs.json` хранятся только агрегаты (`total_seen`, `consecutive_correct`, `interval_days`). Полная история ответов теряется — без неё нельзя будет фитить FSRS-параметры под себя в будущем.

Делать: добавить поле `history: list[dict]` в `Card`. Каждая запись — `{date, delta_days, rating, correct}`. `update_card()` пишет в `history` ДО мутации. ~30 LOC + тесты.

**Почему сейчас, а не потом:** каждый день твоей практики без логирования — потерянные данные. Это работа на накопление; чем раньше начнём логировать, тем больше материала для калибровки потом.

**Когда:** сразу после A.1. Это пассивный режим — идёт в фоне всего остального.

### A.3 — Калибровка SM-2-констант

**Когда:** после ≥ 2 недель данных в A.2.

### A.4 — FSRS-апгрейд (отложено, решение #11 из CLAUDE.md)

### B.1-legacy — фишки: казино-номиналы + центральный пот *(done 2026-05-30)*

**Модель (cash, в BB):** номиналы `0.1 / 0.5 / 1 / 5 / 25`, цвет = номинал (якоря: белый=1, красный=5, зелёный=25; ниже стальной 0.1, голубой 0.5). ### B.4 — Мобильная адаптация *(Этапы 1–3 done 2026-05-29; scope: Drill/Learn/Visualizer)*

*Этап 2 — bottom-sheet настроек (свайп вверх):*
- Левая панель (`.panel-left`) → нижний выдвижной лист: `position:fixed; bottom:0; transform:translateY(calc(100% - 46px))`, по `.sheet-open` → `translateY(0)`. Торчит ручка 46px с лейблом «Settings». **Решение пользователя:** именно bottom-sheet со свайпом вверх, не боковой drawer по гамбургеру.
- Новый модуль `static/mobile.js` (изолированный, vanilla): вешает ручку на все `.panel-left`, открытие свайпом вверх/тапом, закрытие свайпом вниз/тапом по backdrop/сменой вкладки. `matchMedia`-гейт + закрытие при возврате на десктоп.
- **Gotcha зафиксирован:** `position:fixed` у потомка элемента с `transform` позиционируется относительно этого предка, не вьюпорта. Поэтому FAB (ниже) и backdrop живут на `body`, не внутри `.panel-left`.

*Этап 3 — touch + плавающая Deal:*
- **Находка:** кнопка `#newHandBtn` (Deal Hand) физически внутри `.panel-left` → на мобилке пряталась в лист. **Решение пользователя:** плавающая кнопка. `mobile.js` создаёт FAB `.mobile-deal-fab` на `body`, проксирует клик на настоящий `#newHandBtn` (логика drill.js не тронута). FAB виден только в Drill на узком вьюпорте (`updateDealFab` по смене вкладки), `z-index:120` < backdrop(150) → гаснет при открытом листе.
- Touch-таргеты в медиа-блоке: `.action-btn` 48px, `.seg`/`.pos-buttons .seg` 44px, `.fb-next/.fb-hint/.learn-*-btn` 44px, `.mini-select` 40px.

**Cache-bust:** `style.css?v=20260529j`, `drill.js?v=20260529h`, `mobile.js?v=20260529b`.

**Тестирование (сделано):** headless Playwright (chromium) на iPhone SE (375), iPhone 15 (393), Galaxy S20 (360) + десктоп 1280 для регресса. Проверено: нет горизонтального переполнения, стол на экране, 6 мест в пределах стола, матрица не схлопнута, лист открывается/закрывается, FAB раздаёт руку и скрыт вне Drill, на десктопе панель `static`+ручка/FAB скрыты (нулевой регресс).
